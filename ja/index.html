1. 概要

DOORI アプリは、スマートフォンから自動ドアコントローラー（MCU）に直接接続し、ドアの開閉と状態確認を行うアプリです。

  • 通信方式 — 同じ WiFi ネットワーク内で自動ドアコントローラーに直接接続します。
  • ローカル専用 — 自動ドアの WiFi（AP）に接続した状態でのみ動作します。モバイルデータ・外部ネットワーク経由の接続には対応していません。
  • 多言語 — 한국어 · English · 中文 · 日本語。
  • 画面 — テーマ（ダーク/ライト/ホワイト）· 文字サイズ（小/標準/中/大、設定画面のみ）· ボタンサイズ（固定/画面に合わせる）。
重要 — ドアの制御、センサー設定、パスコード変更はすべてコントローラーに接続した状態でのみ動作します。 パスコードとセンサー値はアプリではなくコントローラー（MCU）に保存されます。

接続手順

  1. スマートフォンを自動ドアコントローラーと同じ WiFi に接続します。自動ドアは自身の WiFi を発信しているので、 スマートフォンの WiFi 設定からこの WiFi を選んで接続します。
  2. DOORI アプリを起動します。アプリを新たに起動すると、スタート画面が約 1.4 秒表示された後 ホーム画面（オフライン）が開きます（自動接続はしません）。
  3. ホーム画面右下のシステム領域にある接続ボタンを押します。接続されると パスコード入力（コントローラー認証）画面が表示されます。設置時にお伝えした 4 桁のパスコードを入力してください。 認証後に制御が有効になります。
  4. 接続できない場合は 11. トラブルシューティングをご確認のうえ、解決しない場合はお問い合わせください。
パスコードは接続してドアを制御するときに入力します。

3. ホーム画面

ホーム画面はテーマによって見た目が変わりますが、構成と動作は同じです。上から順に以下の領域があります。

3.1 上部ヘッダー

左側はアプリのアイコンと名称、右側の設定ボタンから設定メニューに入ります。

3.2 ステータスパネル（STATUS）

項目意味
STATUS現在の状態 — 開放中 / 閉鎖中 / 開 / 閉 / 停止 / エラー / 開放（全開保持）/ ロック。表示する状態がない場合は -
RUNTIME最後に動作が発生した時刻（月/日 · 時:分:秒）。動作記録がない場合は -
LINKコントローラーの接続状態 — 接続済み（緑）/ 接続中…（アンバー）/ オフライン（赤）

STATUS の色分け — 進行中（開放中/閉鎖中）と停止は黄色、エラーとロックは赤色、それ以外（開/閉/開放）は緑色です。

動作確認の表示 — 区切り線の点滅
制御ボタンを押して動作が実際に送信された瞬間、ステータスパネル内の縦の区切り線（STATUS | RUNTIME | LINK の間の線）が 一瞬点滅して消えます。特にロックやリセットのような長押しが必要なボタンは指で隠れて動作のタイミングが分かりにくいのですが、 この点滅で「今コマンドが送られた」ことをすぐ確認できます。点滅はコマンドが実際に送信された場合にのみ表示されます。

3.3 センサー検知（SENSOR / SAFETY）

  • SENSOR — 人や物体を検知してドアの開放を発生させるセンサー。
  • SAFETY — 安全センサー。閉まるドアの挟み込みを防ぎます（検知中は閉まりません）。

検知している間はチップが赤く点滅します。瞬間的な検知も見逃さないよう、点滅は最低約 1 秒間保持されます。

3.4 基本制御（4 つのボタン）

接続済みかつロックされていない場合のみ有効になります。

ボタン動作
開く設定された開度までドアを開きます。
閉じるドアを閉じます。
停止動作中のドアをその場で停止します。
開放全開にした後、その状態を保持します。

3.5 開度設定（100 / 50 / 30）

ドアをどこまで開くかの開度をあらかじめ選びます（プリセット 3 つ）。プリセットを押すと開度だけが送られ、 ドアはすぐには動きません — その後開くボタンを押すと、この開度で動作します。

3.6 システム（3 つのボタン）

ボタン動作
ロック / 解除誤タッチ防止のため長押し（既定 1 秒）で動作します（ボタンには ロック(1S) のように表示）。 ドアをロックすると制御ボタンが無効になります。
リセット長押し（既定 1 秒）でコントローラーにリセットコマンドを送ります。短く押しても動作しません。
接続接続のオン・オフを切り替えます。緑色の接続済みが正常な状態です。
ロックとリセットはコントローラーに接続されているときのみ有効です。未接続の状態では グレー表示になり押しても動作しません。接続ボタンだけは常に有効です。

4. コントローラーへの接続

  • 接続 — アプリ起動時に自動接続はせず、ホームの接続ボタンを押して接続します。
  • 認証 — セッション中の最初の接続時に一度パスコードを求められます。一度認証すれば、 同じセッション内では切断・再接続されても再度尋ねられません。
  • 自動再接続 — 接続が切れると自動的に再接続し、先に入力したパスコードで自動的に再認証します。 パスコードはアプリ実行中のみメモリに保持され保存されないため、アプリを終了して起動し直すと再入力が必要です。
  • ローカル専用 — WiFi がオフになると接続を切り、WiFi が戻ると自動的に再接続します。

自動ドア WiFi に接続する — 既定はスマートフォンの WiFi 設定から手動で

出荷時の設定ではアプリが WiFi を自動的に掴むことはありません。スマートフォンの設定 ▸ WiFi から自動ドア WiFi （名称は WizFi360_… のように表示されます）に接続してから、アプリのホームで接続を押してください。 自動ドア WiFi でない状態で接続を押すと案内画面が表示されます — スマートフォンの WiFi を切り替えて再試行を押してください。

アプリから自動で接続する（WiFi自動検索 — Android 10 以上 · iPhone）

設定 ▸ ネットワークで「WiFi自動検索」をオンにし、自動ドアの正確な WiFi 名（例： WizFi360_123456 — スマートフォンの WiFi 一覧で確認）を入力しておくと、接続案内画面に 「(WiFi名) に接続」ボタンが表示されます。スマートフォンの WiFi 設定に移動しなくても、アプリから直接接続できます。

  1. 「(WiFi名) に接続」を押します。
  2. スマートフォンが「デバイスに接続しますか？」と尋ねます → 接続を押します。 （最初の一度だけ表示され、以降はスマートフォンが記憶してすぐ接続します。）
  3. 自動ドア WiFi に接続されると、アプリが自動的にコントローラーまで接続しパスコード画面に進みます。
接続中はスマートフォンの WiFi が自動ドア WiFi に切り替わります。自動ドア WiFi にはインターネットがないため、 その間は他アプリの通信がモバイルデータ（LTE·5G）に切り替わります。データ通信が使えない状態では、 DOORI を使っている間インターネットに接続できません。これは正常な動作で、 アプリを終了するか接続を解除すると元の WiFi に自動的に戻ります。

6. ネットワーク設定

コントローラー WiFi

  • WiFi自動検索（トグル、既定オフ）— オフの場合はスマートフォンの WiFi 設定から直接接続（既定）、 オンにすると下に入力した WiFi 名でアプリが自動接続を試みます。
  • コントローラー WiFi 名 — 自動ドア WiFi の正確な名称（例：WizFi360_123456）。 機器ごとに末尾が異なるため、スマートフォンの WiFi 一覧で確認してそのまま入力してください。
  • WiFi パスワード — 自動ドア WiFi のパスワード（設置時にお伝えした値）。

WiFi 情報は保存されるだけで、現在の接続には影響しません（次回の接続から適用）。周囲に自動ドアが複数あっても、 正確な名称を指定するため入力したその 1 台にのみ接続します。

8. パスコード設定

パスコードはコントローラー（MCU）に保存されます。そのため接続された状態でのみ変更できます。

変更手順（4 桁の数字）：① 現在のパスコードを入力（コントローラーが検証）→ ② 新しいパスコードを入力 → ③ パスコード確認で再入力。

変更が確定すると「パスコードを変更しました」という通知とともにホームに戻り、コントローラーが一時的に再起動するため 円形のカウントダウンタイマーが表示されます。ほとんどの場合、数秒以内に自動的に再接続されます （新しいパスコードで自動認証されるため、再入力は不要です）。

9. 言語 · 画面

言語設定

設定 → 言語設定 → 한국어 / English / 中文 / 日本語 から選択します。選択すると即座に画面全体に反映されます。 インストール直後はスマートフォン（システム）の言語に従い、対応していない言語の場合は英語で表示されます。 一度選択すると、その選択が保持されます。

画面設定（テーマ · 文字サイズ · ボタンサイズ）

  • テーマ — ダーク / ライト / ホワイトの 3 種類。選択すると即座に反映されます。
  • 文字サイズ — 小/標準/中/大の 4 段階。設定画面の文字だけ大きくなり、ホーム画面には影響しません。 段階を変えてもレイアウトが崩れることはありません。
  • ボタンサイズ — 画面に合わせる（既定）：画面の高さに合わせてホームのボタンを大きくし余白を埋めます · 固定：すべての機種で同じ大きさを保ちます。

10. 使用履歴

設定 → 使用履歴からドアの動作履歴を確認します。表示対象：開放/閉鎖完了 · 停止 · エラーなどのドア動作と、 ロック / 解除 / リセットの記録。

使用履歴にはコントローラーで実際に起きた状態変化のみが残ります。定期的な状態確認（ヘルスチェック）、 センサー検知の表示、開度の変更など、画面に見えるものすべてが記録されるわけではありません — 同じ状態が繰り返されて 記録が埋め尽くされるのを防ぐためです。
  • 期間フィルター：今日 / 昨日 / 7 日 / 30 日 · 上部に開放・閉鎖回数の統計。
  • 保存：直近 30 日 + 最大 3,000 件（超過分は自動削除）。記録を消去で全件削除できます。

接続が自動的に戻らないとき — 手動での再接続

DOORI は自動ドア WiFi の中でのみ動作します。そのためスマートフォンがその WiFi から離れると、アプリは バッテリーを節約するために自動再接続を一時的に停止します。故障ではなく、決められた動作です。 このとき LINK 欄はオフラインと表示され、制御ボタンはグレーになりロックされます。

状況アプリの動作
ドアから離れて自動ドア WiFi の圏外になった数回試したあと静かに停止します
モバイルデータ（LTE·5G）に切り替わった同様に停止します（データ通信では自動ドアに到達できません）
他の WiFi（自宅・オフィスなど）に移ったもう少し試したあと停止します
自動ドア WiFi に戻ってきた自動的に再接続します
電波が弱く一時的に切断された自動的に数回再接続を試みます

解決方法 — 2 ステップで済みます。

  1. スマートフォンが自動ドア WiFi に接続されているか確認してください。
  2. ホーム画面右下の 接続 ボタンを押してください。このボタンを押すとアプリが 最初からやり直して接続を試みます — 自動再接続が停止した後でも必ず動作する、最も確実な方法です。
しばらく席を外して戻ったときに接続が見当たらない場合は、お待ちにならず 接続 ボタンを一度押してください。 アプリが自力で再接続するのを待つよりずっと早く復帰します。

12. よくある質問

Q. 複数のスマートフォンから同時に制御できますか？
可能です。複数の端末がそれぞれ接続・認証すれば一緒に使用できます。ただし複数の端末が同時にコマンドを送ると、 実行中のコマンドが優先されて一部のコマンドが無視されるなど、意図と異なる動作に見える場合があります。 ドアを操作する際は一度に 1 台からの操作をおすすめします。
Q. パスコードを忘れてしまいました。
パスコードはコントローラー（MCU）に保存されています。コントローラーの初期化、または設置業者へのお問い合わせが必要です。
Q. 開度 100/50/30 以外の値は設定できませんか？
ホーム画面では 100/50/30 のプリセットのみをご用意しています（スライダーはありません）。
Q. 少し席を外して戻ったら接続が切れていました。故障ですか？
いいえ。スマートフォンが自動ドア WiFi から離れると、アプリはバッテリーを節約するため自動再接続を停止します。 自動ドア WiFi に再接続してから、ホームの接続ボタンを押してください。
Q. アプリを起動したままドアの前に行けば自動で接続されますか？
自動ドア WiFi につながったままであれば、たいていは自動的に再接続されます。ただし途中で他の WiFi や モバイルデータに切り替わっていた場合は、ご自身で 接続 ボタンを押す必要があります。
Q. インターネット（外部）から自宅の自動ドアを開けられますか？
対応していません。DOORI は自動ドアと同じ WiFi（AP）内でのみ接続します。遠隔制御はセキュリティ上の課題があるため、 別の製品構成で扱います。
Q. アプリが収集する個人情報はありますか？
ありません。アカウント・ログイン・会員登録はなく、外部サーバーとも通信しません。 詳しくはプライバシーポリシーをご覧ください。
